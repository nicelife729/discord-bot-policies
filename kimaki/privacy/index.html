Kimaki Privacy Policy

Effective date: July 5, 2026

This Privacy Policy explains what information Kimaki may process when it is added to a Discord server or used by server members. This policy does not replace Discord's own privacy policy or developer terms.

1. Covered Service

This policy applies to Kimaki and related Discord bot services operated for K-Mika community and utility features.

2. Terminology

Server Manager means a person who can add, remove, or configure the bot. Server Member means a person in a server where the bot is installed. User means anyone who interacts with the bot directly or indirectly.

3. Data Provided by Commands

The bot may collect or store data intentionally provided through commands, settings, or interactions, including Discord user IDs, server IDs, channel IDs, role IDs, message IDs, command text, prompts, links, reminders, notes, parameters, server-specific settings, feedback, and support information.

4. Data Collected When Features Are Enabled

If optional features are enabled, the bot may store community utility configuration, reminder or scheduled-message data, channel routing, permission settings, and logs needed to diagnose failed bot actions.

5. Data Processed Automatically

During normal operation, the bot may temporarily process data supplied by Discord's API, including usernames, display names, avatars, IDs, server/channel/message/role metadata, permission metadata, timestamps, command usage metadata, error logs, and message content when needed to process commands, mentions, or enabled features.

6. Data Storage and Retention

Stored data is kept only as long as reasonably needed to provide features, maintain configuration, debug issues, prevent abuse, or comply with platform requirements. No electronic storage method is perfectly secure, but reasonable efforts are made to protect stored data.

7. Data Sharing

The bot does not sell personal data. Data may be shared only when necessary to operate the bot, comply with legal obligations or valid platform requests, or investigate abuse, security incidents, or terms violations.

8. User Choices and Removal Requests

Server managers can remove the bot from a server to stop future processing in that server. To request deletion of stored data associated with a server or user, use the issue tracker of the GitHub repository that hosts this policy page.

9. Changes

This policy may be updated when the bot, data practices, or platform requirements change. Continued use of the bot after an update means you accept the updated policy.
