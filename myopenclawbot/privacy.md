# MyOpenClawBot Privacy Policy

Effective date: July 5, 2026

This Privacy Policy explains what information MyOpenClawBot may process when it is added to a Discord server or used by server members. It follows the practical structure commonly used by Discord bot policies: what is provided by commands, what is collected when features are enabled, what is processed automatically, how data is stored, and how users can request removal.

This policy does not replace Discord's own privacy policy or developer terms.

## 1. Covered Service

This policy applies to MyOpenClawBot and related Discord bot services operated for OpenClaw tool assistance.

## 2. Terminology

- Server Manager: a person who can add, remove, or configure the bot in a Discord server.
- Server Member: a person who is a member of a Discord server where the bot is installed.
- User: anyone who interacts with the bot directly or indirectly.

## 3. Data Provided by Commands

The bot may collect or store data that users intentionally provide through commands, settings, or interactions, including:

- Discord user IDs, server IDs, channel IDs, role IDs, and message IDs needed to execute commands.
- Command text, prompts, links, parameters, and configuration values submitted to the bot.
- Tool workflow settings, task labels, automation preferences, and other bot-specific configuration.
- Feedback, bug reports, or support information voluntarily submitted by users.

Do not submit passwords, authentication tokens, private keys, or other secrets to the bot.

## 4. Data Collected When Features Are Enabled

If a server manager enables optional features, the bot may store data required for those features, such as:

- Server-specific command configuration.
- Channel routing or permission settings.
- Saved workflow templates or automation preferences.
- Logs needed to diagnose failed bot actions.

Optional feature data is collected only when the corresponding feature is used or configured.

## 5. Data Processed Automatically

Like most Discord bots, MyOpenClawBot may temporarily process data supplied by Discord's API during normal operation, including:

- Usernames, display names, avatars, and IDs.
- Server, channel, message, role, and permission metadata.
- Timestamps, command usage metadata, and error logs.
- Message content only when Discord provides it to the bot and it is needed to process commands, mentions, or enabled features.

Some of this data may be cached in memory while the bot is running. Cache data is used for normal bot operation and may be cleared when the bot restarts.

## 6. Data Storage and Retention

Stored data is kept only as long as reasonably needed to provide bot features, maintain server configuration, debug issues, prevent abuse, or comply with platform requirements.

Operational logs may be retained for a limited period for debugging and security. Server configuration may remain stored until a server manager changes it, disables the feature, or requests removal.

No method of electronic storage is perfectly secure, but reasonable efforts are made to protect stored data from unauthorized access.

## 7. Data Sharing

The bot does not sell personal data. Data may be shared only when necessary to:

- Operate the bot or hosting infrastructure.
- Comply with legal obligations or valid platform requests.
- Investigate abuse, security incidents, or violations of the Terms of Service.

The bot may rely on Discord and hosting providers, which have their own privacy practices.

## 8. User Choices and Removal Requests

Server managers can remove the bot from a server to stop future processing in that server. Users can avoid using bot commands if they do not want to provide command data.

To request deletion of stored data associated with a server or user, use the issue tracker of the GitHub repository that hosts this policy page. Include enough non-sensitive information to identify the relevant server, user, or configuration.

## 9. Changes

This policy may be updated when the bot, data practices, or platform requirements change. Continued use of the bot after a policy update means you accept the updated policy.
